**Welcome from DevRelCon 2026!** If you're reading this, we probably just shared a morning heartbeat drum together.

## Quickstart

```bash
# 1. Fork the repository
gh repo fork timfalls/soul.md

# 2. Navigate to your local copy
cd soul.md

# 3. Open SOUL.md and configure your own personal root file - a reference point to guide your work so that it feels aligned with who you are.
```

> **System Architecture:**  
> Your `SKILLS.md` file reflects the output of your work (posts, talks, apps, tutorials), while your `SOUL.md` file is a reflection of the inputs, resources, and wiring required for you to deploy those skills successfully and sustainably (for your community, company, and self).


## What's in this Repository?

1. **`SOUL.md`**: Your configuration template. This is where you define your Core Values, Standards, Dependencies, Read-Only boundaries, and Error Handling protocols.
2. **`.env.example`**: A space to acknowledge your current external context (e.g., `INDUSTRY_MODE="ai_wave"`). You can't control the environment variables, but you can configure how your system responds to them.
3. **`ACKNOWLEDGEMENTS.md`**: A dedicated file for honoring teachers, ancestors, lineage, and land—grounding the work in gratitude before execution.
4. **`LICENSE`**: Open-source MIT License, inviting you to fork, adapt, and share your own version freely.

## How to Use This Template

1. **Fork this repository** to your own GitHub account.
2. **Make your fork private** (your `SOUL.md` is for you first and foremost, I encourage you to start in private mode, to encourage honesty with yourself. Later, you might choose to open-source it, as a sign of transparency with your community and/or team.
3. Open the `SOUL.md` file, create an environment for you to be introspective, and start defining your root configuration. Journaling can be a helpful way to discover your values, standards, boundaries, dependencies, etc. 
4. Rename `.env.example` to `.env` and log the current seasons/variables you are operating under.
5. **Commit your changes.** 
6. **Return to it regularly.** Like any well-maintained system, your `SOUL.md` requires discipline, practice, and consistency to run effectively. Consider making this a ritual you return to periodically.
