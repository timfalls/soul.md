# soul.md: The Root Configuration File for DevRel

**Welcome from DevRelCon 2026!** If you're reading this, we probably just shared a morning heartbeat drum together.

## The Public Product vs. The Internal Logic

In DevRel, we spend a lot of energy polishing our `README.md`. The talks we give, the content and code we ship, the communities we build, and the brand we present to the world are our public, generally available product.

But a `README.md` is fragile if the internal logic running underneath it is broken. 

That's why this repository is here for you. It's a dedicated space to configure your **`SOUL.md`**—the root system configuration file that dictates *how* and *why* you execute your skills, especially amidst the chaotic environment of the current AI wave (and the world in general, really).

## What's in this Repository?

1. **`SOUL.md`**: Your blank configuration template. This is where you will define your Core Values, Standards, Dependencies (your "medicine spirit"), Read-Only boundaries, and Error Handling protocols.
2. **`.env.example`**: A space to acknowledge your current external context (e.g., `INDUSTRY_MODE=ai_wave` or `CURRENT_CAPACITY=75%`). You can't control the environment variables, but you can configure how your system responds to them.

## How to Use This Template

1. **Fork this repository** to your own GitHub account.
2. **Make your fork private** (your `SOUL.md` is for you and you alone, unless you actively choose to open-source it).
3. Open the `SOUL.md` file and start defining your root configuration.
4. Rename `.env.example` to `.env` and log the current seasons/variables you are operating under.
5. **Commit your changes.** 
6. **Return to it regularly.** Like any well-maintained system, your `SOUL.md` requires discipline, practice, and consistency to run effectively.
